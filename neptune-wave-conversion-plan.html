Neptune to WAVE-style conversion plan

The current Neptune application is already a strong foundation. It contains the core logic for site assessment, water balance, treatment selection, financial evaluation and sustainability value creation. The opportunity is to package that same logic into a more guided and product-like experience, similar to the DuPont WAVE model.

1. What can be extracted from the current app

  • Site and regulatory context: country-specific limits, water stress and climate context.
  • Process model: flow balance, water sources and contamination inputs.
  • Design engine: treatment train generation, removal chain and compliance check.
  • Economic engine: CapEx, OpEx, payback, EBITDA bridge and value creation output.
  • Visualisation layer: plots, summary metrics and flow diagrams.

2. How to align it more closely with the WAVE approach

  • Make the app feel like a single guided workflow rather than a set of standalone pages.
  • Introduce a design assistant that recommends a starting treatment concept based on the input profile.
  • Group technologies into a technology library with clear capability, cost and performance attributes.
  • Present a scenario comparison screen for reuse rate, treatment option and financial trade-offs.
  • Move from “analysis” to “design software” by adding robust output exports and report-style summaries.

3. Suggested product structure

  1. Define the site — country, industry, water stress and discharge limits.
  2. Model the water system — source flows, process balances and contaminants.
  3. Select the treatment train — pre-treatment, biological, polishing and concentrate handling.
  4. Evaluate economics — CapEx, OpEx, payback, value creation and sensitivity.
  5. Generate the report — printable or downloadable design summary.

4. Practical implementation recommendations

  • Keep the current Streamlit structure as the calculation engine.
  • Wrap it in a more polished landing experience with a hero section, benefit cards and a clear “start design” entry point.
  • Rename the current modules into product-style labels such as Site Setup, Water Model, Treatment Design and Business Case.
  • Introduce default templates for common industries such as steel, food, mining and chemicals.
  • Use one unified navigation bar and one consistent design language throughout the app.

5. Recommended next steps

  • Deploy the current Streamlit app to a public URL.
  • Build a polished landing page and link it to the app.
  • Refactor the pages into a guided workflow with a single start button.
  • Add scenario comparison and a downloadable report output.
  • Use the DuPont WAVE page as a visual and messaging reference, but keep the brand and content aligned with your own professional positioning.
