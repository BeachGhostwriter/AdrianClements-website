Neptune to WAVE-style conversion plan

The current Neptune application is already a strong foundation. It contains the core logic for site assessment, water balance, treatment selection, financial evaluation and sustainability value creation.

1. What can be extracted from the current app

  • Site and regulatory context: country-specific limits, water stress and climate context.
  • Process model: flow balance, water sources and contamination inputs.
  • Design engine: treatment train generation, removal chain and compliance check.
  • Economic engine: CapEx, OpEx, payback, EBITDA bridge and value creation output.

2. How to align it more closely with the WAVE approach

  • Make the app feel like a single guided workflow rather than a set of standalone pages.
  • Introduce a design assistant that recommends a starting treatment concept based on the input profile.
  • Group technologies into a technology library with clear capability, cost and performance attributes.
  • Present a scenario comparison screen for reuse rate, treatment option and financial trade-offs.

3. Suggested product structure

  1. Define the site — country, industry, water stress and discharge limits.
  2. Model the water system — source flows, process balances and contaminants.
  3. Select the treatment train — pre-treatment, biological, polishing and concentrate handling.
  4. Evaluate economics — CapEx, OpEx, payback and value creation.
  5. Generate the report — downloadable design summary.
